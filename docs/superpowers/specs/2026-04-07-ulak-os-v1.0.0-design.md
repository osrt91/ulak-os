# Ulak OS v1.0.0 — Design Spec

**Tarih:** 2026-04-07
**Yazar:** Brainstorming oturumu (Claude + kullanıcı)
**Durum:** Onay bekliyor
**Hedef teslim:** Bugün
**Sonraki adım:** `superpowers:writing-plans` skill'i ile implementation plan üretimi

---

## 1. Bağlam

Mevcut workspace `claudeulak.github/`, "Claude Ulak 1.9.1" iç sürüm serisinin son halini barındırıyor. İçerikte üç vendor (Claude Code, Codex/Copilot, Gemini CLI) için adapter dosyaları, 20 subagent, 5 slash command, 4 native skill, governance dokümanları ve dağıtım paketleri var. `/doctor` analizinde tespit edilen sorunlar:

- `Claude Ulak` markası vendor-spesifik (codex/gemini kullanıcıları için kafa karıştırıcı)
- İki yığılmış klasör (`_equal_distribution_pack`, `_equalized_github_repo`) + üst kökte dağınık dosyalar
- `.claude/settings.local.json` geçersiz JSON (trailing comma)
- Üst kök `.claude/settings.json` her şeye allow veriyor (repo profilini bypass ediyor)
- Versiyon tutarsızlığı: depo 1.9.1 ama core contract dosyası `1.9.0`
- `.mcp.json` env var'ları dokümante edilmemiş
- Pack vs repo ilişkisi belirsiz
- Core contract 12 artefakt sayıyor, AGENTS.md sadece 8 zorunlu listeliyor (cross-vendor davranış farkı riski)
- `version-lineage.md` 1.9.1'i göstermiyor
- `reports/current/` dizini gitkeep'siz (klonlayanda eksik olur)

Kullanıcı bu projeyi bugün GitHub'a public repo olarak yayınlamak ve kendisi test etmek istiyor.

## 2. Hedefler

1. Markayı vendor-nötr hale getir: **Ulak OS** (repo: `ulak-os`)
2. Versiyonu sıfırla: **1.0.0** ("first stable public release under Ulak OS brand")
3. Tek kanonik repo dizini oluştur, eski içeriği koru
4. Klonlayan kişi 5 dakikada ilk komutu çalıştırabilsin
5. CI hatalı commit'leri yakalasın (broken @import, geçersiz JSON, brand kalıntısı, secret leak)
6. Üç vendor için kurulum bootstrap scriptleri sağla (Win + macOS + Linux)
7. Public superpowers ekosistemiyle temiz entegrasyon (mapping doc + 1 PoC wrapper)
8. Çok dilli yapı (TR + EN bugün, gelecek diller için strüktür hazır)
9. Tüm `/doctor` bulgularını çöz

## 3. Hedef olmayanlar (v1.0.0 dışında, v1.1+ için kayıtlı)

- 6 dil çeviri (FR, DE, ES, AR, JA, ZH)
- 5 ek superpowers wrapper komutu (`/ulak-roadmap`, vb.)
- Smoke test matrix (3 vendor × 5 komut)
- Rule collision matrix'i deterministic hook'a çevirme
- CODEOWNERS, issue/PR şablonları, GitHub discussions
- Plugin marketplace yayını
- Eval/golden prompt regresyon seti
- Çalıştırılabilir demo projeler (`examples/`)
- v1.0.0'dan önceki sürüm artefaktlarının (`releases/1.0.0`... `releases/1.9.1`) yeniden yapılandırılması

## 4. Başarı kriterleri

| # | Kriter | Doğrulama |
|---|---|---|
| 1 | Klonlayan kişi 5 dk içinde ilk komutu çalıştırabilir | README → init script → vendor CLI → ilk komut akışı bash session'da test edilir |
| 2 | CI yeni commit'te kırmızıya düşer (broken commit) | Test PR'ı oluştur, kasıtlı broken @import ekle, CI fail etmeli |
| 3 | Brand tutarlılığı | `grep -rn "Claude Ulak" ulak-os/` boş döner |
| 4 | TR + EN paralel dokümantasyon | README, CLAUDE/AGENTS/GEMINI, core contract her iki dilde mevcut |
| 5 | Tüm `/doctor` bulguları çözülmüş | Her madde için commit/dosya referansı |

### 5.2 Ana mimari kararlar

- **Sibling dizin, üst kök dokunulmaz**: `ulak-os/` üst workspace içinde sibling olarak doğar. Eski içerik silinmez, kaynak/yedek olarak kalır. Sadece `ulak-os/` git'lenir.
- **TR birincil, EN paralel**: Birincil dosyalar TR (uzantısız), EN ekler `.en.md` uzantısıyla. Gelecekte 3. dil için `.fr.md`, `.de.md` gibi şablon kurulu.
- **Vendor adapter dosya isimleri korunur**: `CLAUDE.md`, `AGENTS.md`, `GEMINI.md` — bunlar vendor sözleşmesi, marka değil.
- **Core contract version reset**: `claude-ulak-core-contract-1.9.0.md` → `ulak-os-core-contract-1.0.0.md`. Hem isim hem versiyon hizalı.
- **Eski klasörler dokunulmaz**: Mevcut `claude-ulak_1.9.1_*` klasörleri ve dosyaları workspace kökünde olduğu yerde kalır. `ulak-os/` git'lemesine girmezler. Kullanıcı sonradan istediği zaman silebilir.

## 6. Bileşenler

### 6.1 Rebrand (mekanik)

- Global replace: `Claude Ulak` → `Ulak OS`, `claude-ulak` → `ulak-os`, `claude_ulak` → `ulak_os`
- **Korunan referanslar** (whitelist):
 - Vendor sözleşme dosyaları: `CLAUDE.md`, `AGENTS.md`, `GEMINI.md`
 - Vendor adapter docs: `claude-code.md`, `codex-cli.md`, `gemini-cli.md`
 - Vendor config dizinleri: `.claude/`, `.gemini/`
 - Internal codename map (V7↔1.4.0 gibi tarihsel kayıtlar)
 - URL/link metni: claude.com, github.com referansları
- Versiyon: tüm `1.9.1` referansları `1.0.0`'a dönüşür, ek olarak `version-lineage.md`'ye geçiş paragrafı eklenir
- Core contract dosya rename + versiyon header güncellemesi

### 6.2 Bootstrap scriptleri (kurulum otomasyonu)

Her vendor için iki dosya: `.sh` (POSIX) + `.ps1` (PowerShell). Toplam 6 dosya.

**Ortak iskelet (her script ~50 satır):**
1. Banner: "Ulak OS → <vendor> init"
2. Vendor binary check (`command -v` / `Get-Command`)
3. Required file check (CLAUDE.md veya AGENTS.md veya GEMINI.md, core contract)
4. Dizin oluşturma: `reports/current/`
5. Opsiyonel MCP env var uyarısı (warn, fail değil)
6. "İlk komut" rehberi (vendor'a göre)

**Vendor-spesifik kontroller:**
| Script | Binary | Required files | İlk komut |
|---|---|---|---|
| init-claude | `claude` | CLAUDE.md, prompts/core/ulak-os-core-contract-1.0.0.md | `claude` → `/memory` → `/director komple` |
| init-codex | `codex` veya `copilot` | AGENTS.md,.github/copilot-instructions.md | "Read AGENTS.md, CLAUDE.md,..." |
| init-gemini | `gemini` | GEMINI.md,.gemini/commands/*.toml | `gemini` → `/memory reload` → `/commands reload` → `/director komple` |

**Hata kuralları:**
- Set-strict mode (`set -euo pipefail` / `$ErrorActionPreference = 'Stop'`)
- Exit kodları: 0=ok, 1=fatal, 2=warned
- Hata mesajları kullanıcı dostu (link içerir)

### 6.3 CI — kalite kapısı

**`.github/workflows/ci-validation.yml`** (ana workflow):
```yaml
name: ci-validation
on:
 push: { branches: [main] }
 pull_request: { branches: [main] }
jobs:
 validate:
 runs-on: ubuntu-latest
 steps:
 - uses: actions/checkout@v4
 - name: Validate JSON/TOML schemas
 run: bash scripts/validate-schemas.sh
 - name: Validate @import chains
 run: bash scripts/validate-imports.sh
 - name: Brand consistency
 run: |
 if grep -rn "Claude Ulak" --include="*.md".; then
 echo "ERROR: 'Claude Ulak' kalıntısı bulundu (rebrand eksik)"
 exit 1
 fi
```

**`.github/workflows/secret-scan.yml`** (ayrı workflow, paralel):
```yaml
name: secret-scan
on: [push, pull_request]
jobs:
 gitleaks:
 runs-on: ubuntu-latest
 steps:
 - uses: actions/checkout@v4
 with: { fetch-depth: 0 }
 - uses: gitleaks/gitleaks-action@v2
 env:
 GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**`scripts/validate-imports.sh`**: Tüm `.md` dosyalarını tarar, `^@(.+\.md)$` kalıbını bulur, hedef dosyanın varlığını doğrular. Hatalar `dosya:satır` formatında raporlanır.

**`scripts/validate-schemas.sh`**: Python ile JSON/TOML parse:
- `python -m json.tool` → `.claude/settings.json`, `.mcp.json`
- `python -c "import tomllib; tomllib.load(open(f, 'rb'))"` → `.gemini/commands/*.toml`

### 6.4 Public skill entegrasyonu

**Katman 1: Mapping doc** (`docs/skills-integration/superpowers-mapping.md`)

| Ulak artefaktı | Eşdeğer superpowers skill | Ne zaman |
|---|---|---|
| intake | `superpowers:brainstorming` | GREENFIELD, EXTEND |
| evidence-register | `superpowers:systematic-debugging` | REPAIR, RESCUE |
| execution-roadmap | `superpowers:writing-plans` | Her mod |
| execution | `superpowers:executing-plans` + `test-driven-development` | Execution fazı |
| validation-plan | `superpowers:verification-before-completion` | Her mod (zorunlu) |
| manager-verdict | `superpowers:finishing-a-development-branch` | Her mod (kapanış) |

**CLAUDE.md** ve **GEMINI.md**'ye tek satır eklenir:
> Superpowers marketplace yüklüyse, `docs/skills-integration/superpowers-mapping.md` eşleştirmesini kullan.

**Katman 2: Proof-of-concept wrapper** (`/.claude/commands/ulak-intake.md`)

Tek bir wrapper komut yazılır — diğerleri için şablon görevi görür:

```markdown
---
name: ulak-intake
description: Ulak intake artefaktı üretir; superpowers:brainstorming yüklüyse onu kullanır
---

# Ulak Intake

1. Eğer `superpowers:brainstorming` yüklüyse:
 - O skill'i çağır
 - Çıktısını `reports/current/intake.md`'ye intake formatında kaydet
2. Yüklü değilse:
 - Native intake akışını kullan (proje bağlamı + intent + sınırlar)
 - Aynı dosyayı yaz
3. `inventory` adımına geçişi öner

Bu komut, /ulak-roadmap, /ulak-evidence, /ulak-validate gibi diğer wrapper'lar için şablondur.
```

Kalan 5 wrapper v1.1'e ertelenir.

### 6.5 Çok dilli yapı (TR + EN)

**Strateji**: birincil dosya TR (uzantısız), paralel EN dosyası `.en.md` uzantısıyla aynı dizinde durur. Gelecek diller için `.fr.md`, `.de.md` şablonu hazır.

**EN'ye çevrilen dosyalar (v1.0.0 kapsamı):**
- `README.md` → `README.en.md`
- `CLAUDE.md` → `CLAUDE.en.md`
- `AGENTS.md` → `AGENTS.en.md`
- `GEMINI.md` → `GEMINI.en.md`
- `prompts/core/ulak-os-core-contract-1.0.0.md` → `.en.md`
- `docs/examples/sample-*.md` → `.en.md` (3 örnek)
- `docs/skills-integration/superpowers-mapping.md` → `.en.md`

**EN'ye çevrilmeyen dosyalar (v1.0.0 dışında):**
- 20 subagent dosyası, 5 command, 4 native skill, governance docs, version lineage
- Gerekçe: hacim ve disiplinli çeviri sürelerini bugün karşılayamayız
- v1.1'de "tüm dokümantasyon EN" hedefiyle ele alınır

**Dizin yapısı**: `docs/i18n/{tr,en}/` placeholder dizinleri oluşturulur (gelecek diller için strüktür hazır olsun, v1.1+'da doldurulur).

### 6.6 Paralel ajan kullanımı (execution sırasında)

Implementation fazında `superpowers:dispatching-parallel-agents` skill'i kullanılır. Bağımsız iş parçaları paralel subagent'lara verilir:

**Paralel grup A** (rebrand + temel dosyalar):
- Sub-agent 1: dosya kopyalama + global replace (rebrand)
- Sub-agent 2: core contract rename + import güncelleme
- Sub-agent 3: version-lineage.md + RELEASE_NOTES güncellemesi

**Paralel grup B** (yeni dosyalar + EN):
- Sub-agent 4: 6 bootstrap script (3 sh + 3 ps1)
- Sub-agent 5: CI workflows + validate scriptleri
- Sub-agent 6: EN çevirileri (README, CLAUDE, AGENTS, GEMINI)

**Paralel grup C** (örnekler + skill entegrasyon):
- Sub-agent 7: 3 sample artifact (TR + EN)
- Sub-agent 8: superpowers mapping doc (TR + EN) + /ulak-intake wrapper

**Sıralı**: artefakt zinciri tutarlılığı (core ↔ AGENTS) — paralel değil, manuel kontrol gerektirir.

Tahmini süre kazancı: ~%60 (sırasal vs paralel).

## 7. Data flow

### 7.1 Klonlayan kişinin akışı (5-dakika hedef)

```
1. git clone https://github.com/<user>/ulak-os
2. cd ulak-os
3. README.md aç → vendor seç
4. bash scripts/init-claude.sh (veya.ps1)
 ├─ binary check → OK
 ├─ file check → OK
 ├─ MCP env warn (opsiyonel)
 ├─ reports/current/ ready
 └─ "İlk komut: claude → /director komple"
5. claude
6. /memory (CLAUDE.md yüklü mü doğrula)
7. /director komple
8. reports/current/ altında artefaktlar oluşur
```

### 7.2 CI flow

```
push/PR → ci-validation workflow tetiklenir
 ├─ Job 1: validate-schemas
 ├─ Job 2: validate-imports
 └─ Job 3: brand-consistency

push/PR → secret-scan workflow tetiklenir (paralel)
 └─ Job: gitleaks
```

Herhangi bir job kırmızıya düşerse PR mergeable değil.

## 10. Risk tablosu

| Risk | Olasılık | Etki | Azaltma |
|---|---|---|---|
| Rebrand bazı dosyalarda kaçar | Orta | CI yakalar | Brand check job |
| EN çeviri kalitesi düşük olur | Düşük | Brand zarar görür | Sadece kritik dosyalar çevrilir, kullanıcı gözden geçirir |
| CI ilk koşuda yanlış pozitif | Düşük | PR bloklanır | Her job ayrı, fail noktası net |
| Bootstrap PowerShell execution policy | Orta | Windows kullanıcı takılır | README'de `-ExecutionPolicy Bypass` belgesi |
| Eski klasörler yanlışlıkla commit edilir | Düşük | Repo şişer | `git init` sadece `ulak-os/` içinde |
| gitleaks false positive | Düşük | PR bloklanır | `.gitleaks.toml` allowlist eklenebilir |
| /ulak-intake wrapper yanlış imza | Düşük | PoC çalışmaz | Wrapper sadece bir, hızlıca düzeltilebilir |
| Paralel ajanlar arasında dosya çakışması | Orta | Subagent'lar aynı dosyayı yazar | Görev paylaşımı dosya sınırına göre yapılır, çakışan dosyalar sıralı işlenir |
