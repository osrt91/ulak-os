# Ulak OS v2.0.0 — Design Spec

**Tarih:** 2026-04-09
**Yazar:** Brainstorming oturumu (Claude + kullanıcı)
**Durum:** Onaylandı
**Sonraki adım:** `superpowers:writing-plans` skill'i ile implementation plan üretimi

---

## 1. Bağlam

Ulak OS v1.0.0, 50 bin satırlık prompt birikimine sahip, vendor-neutral bir prompt operating system olarak yayınlandı. 20 subagent, 7 command, 4 native skill, 3 vendor adapter (Claude Code, Codex/Copilot, Gemini CLI), CI kapıları ve TR+EN i18n desteği içeriyor. Tamamen markdown/YAML/JSON/TOML dosyalarından oluşuyor — çalıştırılabilir kod yok.

v1.0.0 gayet iyi çalışıyor. v2'nin amacı bu birikimi otomasyona bağlayıp daha verimli hale getirmek.

## 2. Hedefler

1. 50k satır prompt birikimini bir `ulak` CLI aracına sarmak
2. Proje hafızası eklemek (session'lar arası öğrenilenleri persist etmek)
3. Vendor CLI'ları orkestra eden bir katman kurmak
4. `ulak init` ile onboarding'i 30 saniyeye düşürmek
5. Pack sistemi ile prompt versiyonlamayı otomatikleştirmek
6. npm ile global kurulum sağlamak

## 3. v1'den devralınan borçlar (v2 kapsamına dahil)

### 6.1 Katmanlar

| Katman | Ne yapar |
|--------|----------|
| **CLI** | Kullanıcının dokunduğu yüzey. `ulak <komut>` ile çalışır |
| **Core Engine** | Router (intent → intervention mode), artefakt zinciri yönetimi, hafıza |
| **Vendor Adapter** | Hangi AI CLI çalışıyorsa ona prompt'u iletir, çıktıyı toplar |
| **Prompt Pack** | Mevcut 50k satırlık birikim — dokunulmaz, versiyonlanır |

### 6.2 Temel mimari kararlar

- Prompt'lar **paket** olarak korunur — CLI onları orkestra eder, değiştirmez
- SQLite ile proje hafızası — her `ulak run` sonrası öğrenilenler persist eder
- Vendor detect otomatik — `.claude/` varsa Claude, `GEMINI.md` varsa Gemini
- CLI kendi başına AI çalıştırmıyor — vendor CLI'ların üstüne oturan orkestrasyon katmanı

## 7. CLI Komutları

```
ulak <komut> [alt-komut] [bayraklar]
```

| Komut | Ne yapar | Örnek |
|-------|----------|-------|
| `ulak init` | Projeye Ulak OS kurar, vendor detect eder, `ulak.config.json` oluşturur | `ulak init --vendor claude` |
| `ulak run <artefakt>` | Belirli bir artefaktı üretir (vendor CLI'ı tetikler) | `ulak run intake` |
| `ulak run all` | Tam artefakt zincirini sırayla koşar | `ulak run all` |
| `ulak status` | Artefakt zincirinin durumunu gösterir | `ulak status` |
| `ulak validate` | CI kapılarını yerel koşar (schema, import, brand) | `ulak validate` |
| `ulak memory list` | Proje hafızasındaki kayıtları listeler | `ulak memory list` |
| `ulak memory search` | Hafızada arama yapar | `ulak memory search "auth"` |
| `ulak config` | Ayarları gösterir/değiştirir | `ulak config set vendor gemini` |
| `ulak upgrade` | Prompt pack'i son sürüme günceller | `ulak upgrade` |
| `ulak export` | Artefaktları PDF/HTML/JSON olarak dışa aktarır | `ulak export --format pdf` |

## 8. Hafıza Katmanı (Memory Store)

### 8.1 Dizin yapısı

```
.ulak/
├── memory.db          ← SQLite — tüm hafıza burada
├── sessions/          ← her run'ın ham çıktısı (JSON)
│   ├── 001.json
│   └── 002.json
└── config.local.json  ← kullanıcıya özel ayarlar (.gitignore'da)
```

### 8.2 SQLite şeması

```sql
CREATE TABLE sessions (
  id          INTEGER PRIMARY KEY,
  project     TEXT NOT NULL,
  vendor      TEXT NOT NULL,
  command     TEXT NOT NULL,
  started_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
  finished_at DATETIME
);

CREATE TABLE artefacts (
  id          INTEGER PRIMARY KEY,
  session_id  INTEGER REFERENCES sessions(id),
  name        TEXT NOT NULL,
  content     TEXT,
  hash        TEXT,
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE learnings (
  id          INTEGER PRIMARY KEY,
  session_id  INTEGER REFERENCES sessions(id),
  category    TEXT,
  summary     TEXT NOT NULL,
  detail      TEXT,
  tags        TEXT,
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE VIRTUAL TABLE learnings_fts USING fts5(
  summary, detail, tags,
  content='learnings', content_rowid='id'
);
```

### 8.3 Çalışma prensibi

- Her `ulak run` bir session açar
- Vendor CLI çıktısı parse edilir → artefaktlar + öğrenilenler ayrılır
- FTS5 ile tam metin arama
- Hafıza proje bazlı (`.ulak/memory.db`)
- `.ulak/` klasörü `.gitignore`'da — kişisel hafıza repo'ya girmez
- `ulak memory export/import` ile projeler arası taşınabilir

## 9. Vendor Adapter Sistemi

### 9.1 Interface

```typescript
interface VendorAdapter {
  name: string;
  detect(): boolean;
  sendPrompt(prompt: string): Promise<string>;
  isInstalled(): boolean;
  getVersion(): string;
}
```

### 9.2 Detect sırası

```
1. ulak.config.json'da vendor belirtilmiş mi? → kullan
2. .claude/ klasörü var mı?               → Claude Code
3. AGENTS.md + .github/copilot-instructions.md? → Codex
4. .gemini/ klasörü var mı?               → Gemini
5. Hiçbiri → ulak init sırasında sor
```

### 9.3 Adapter ne yapar, ne yapmaz

| Yapar | Yapmaz |
|-------|--------|
| Vendor CLI'ı subprocess olarak çalıştırır | Vendor API'sine doğrudan bağlanmaz |
| Prompt'u vendor formatına sarar | Prompt içeriğini değiştirmez |
| Çıktıyı parse edip artefakt çıkarır | Vendor'a özel özellik eklemez |
| CLI kurulu mu / versiyonu kontrol eder | Vendor hesap yönetimi yapmaz |

## 13. Risk tablosu

| Risk | Olasılık | Etki | Azaltma |
|------|----------|------|---------|
| better-sqlite3 native build sorunları (Windows) | Orta | Kurulum başarısız | Fallback: sql.js (pure JS) |
| Vendor CLI çıktı formatı değişir | Düşük | Adapter bozulur | Adapter'da versiyon kontrolü |
| Pack güncelleme kullanıcı dosyalarını ezer | Düşük | Veri kaybı | Pack sadece `prompts/` dokunur |
| FTS5 büyük hafızada yavaşlar | Düşük | UX kötüleşir | Index limiti + pagination |
| Vendor CLI subprocess güvenlik riski | Düşük | Injection | Input sanitization |
