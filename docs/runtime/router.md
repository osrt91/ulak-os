# Runtime Router

## Purpose
Decide how the system should engage before deep work begins.

## Decisions
1. Project state: greenfield | brownfield | hybrid
2. Intervention mode: CREATE | REPAIR | EXTEND | REFACTOR | MIGRATE | RESCUE | REPACKAGE
3. Program depth: compact | expanded | full-office
4. Surfaces: customer | admin | public API | backend | db | infra | mobile | web | marketing | payments | localization | compliance

## Full-intent rule
If the user intent is clearly full, complete, or end-to-end, do not restart with menu loops. Start artefacts.
