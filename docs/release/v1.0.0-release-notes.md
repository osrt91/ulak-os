# Ulak OS v1.0.0 — Public General Availability

**Release date:** 2026-04-21
**Type:** Major release (semantic reset from internal v2.4.0)
**License:** MIT

Ulak OS is a vendor-neutral **prompt operating system** for Claude Code / Codex / Gemini CLI. This is its first release outside the operator's hands — the first version a stranger can clone, install, use, and extend without asking the author.

## What v1.0.0 is

Ulak OS v1.0.0 (public GA) is the production-quality, externally-supportable baseline. Everything an outside developer needs to go from `git clone` to a first audit to a first scaffolded project is in place:

- **MIT license** — you can use Ulak OS in commercial projects, no strings
- **Bilingual documentation** — README.md (Turkish primary) + README.en.md (English parity); docs/runbooks/ + docs/FAQ.md + user manual (TR + EN)
- **One-liner installers** — `scripts/install.sh` (POSIX) and `scripts/install.ps1` (Windows PowerShell); both idempotent, dry-run-capable, no sudo required
- **Community surface** — CONTRIBUTING.md, CODE_OF_CONDUCT.md (Contributor Covenant v2.1), 3 issue templates, PR template
- **Self-serve governance** — 20+ governance docs define how every decision is made (rule collision, hook boundaries, MCP allowlist, prompt supply chain, pattern-import ledger, evidence trust scoring)
- **Adversarial-tested** — 5-agent final pass (QA, security red team, release readiness, cartography, customer persona) verified the repo against shipping criteria

## What Ulak OS does

Three flagship operations, each invocable as a Claude Code slash command:

### `/director komple` — deterministic full-spectrum audit

A Phase 0 → Phase 5 autonomous director protocol dispatches specialist agents in parallel and produces 15 artefact types per run. Deep inventory (file:line citations, not folder listings). Did-you-know layer (non-obvious findings, not just what you asked). Validation gates (no "done" without evidence). Trust-tiered evidence (T0 live-probe → T1 primary source → T7 AI-inferred unverified). Optional live probes in Phase 4.5.

Outcome: `manager-verdict.md` with `signoff_status: ready | ready-with-residual | blocked` and a validated execution roadmap.

### `/ulak-scaffold` — deterministic full-stack SaaS scaffolder

From `ulak-scaffold <project> --sector payment-integrated-saas --stack nextjs-supabase`, you get:

- Next.js 15 + TypeScript + Tailwind + Supabase SSR
- RLS policies enabled on every user-facing table
- Audit log + tenant isolation + FK integrity
- CI workflow (validate + gitleaks + e2e) — all blocking, no `continue-on-error`
- Preflight script (fast mode <20s; full mode mirrors CI 1-for-1)
- Webhook-triggered deploy with single-flight flock guard + post-deploy health probe + automated rollback
- VPS hardening template (SSH port 2244, key-only, root disabled, UFW, fail2ban)
- Master design contract + per-page override pattern baked in
- Master + per-page design system contract via `DESIGN.md.template`
- 27 templates total, deterministic from commit 1

### `/director persona=<list>` — multi-persona audit

The same director run, but dispatched through 4 customer surfaces (customer · admin · bayi/partner · support) or 6 depending on project shape. Per-persona findings are merged with contradiction-as-residual-risk handling. Uncovers surface-specific UX, security, and governance gaps that a single-role audit misses.

## Why v1.0.0 (and not v3.0.0)

v1.0.0 is a semantic reset. Internally, the repo went through a v1.x → v2.x lineage while under operator control. v1.0.0 (public GA) marks the first release where:

- The license is permissive and explicit
- Documentation is bilingual and written for a stranger
- No operator-internal details leak through public docs
- Install is a one-liner, not a manual workflow
- Every specialist agent is a production 80–110-line spec (not a 31-line stub)
- Community files (CONTRIBUTING, CODE_OF_CONDUCT, issue/PR templates) are in place
- Plugin marketplace submission prep is complete

A stranger adopting Ulak OS today depends on v1.0.0 going forward. The internal v2.x history is preserved in `docs/history/version-lineage.md` for those who want to understand the trajectory.

## Install

POSIX (macOS / Linux):

```bash
curl -fsSL https://raw.githubusercontent.com/osrt91/ulak-os/main/scripts/install.sh | sh
```

Windows (PowerShell 5.1+):

```powershell
iwr -useb https://raw.githubusercontent.com/osrt91/ulak-os/main/scripts/install.ps1 | iex
```

Then in any project:

```bash
cd /path/to/your-project
ulak init .
# Open in Claude Code, run: /director komple
```

See `docs/runbooks/install-methods.md` for alternative install paths (clone, submodule, plugin marketplace, npm — the latter deferred to post-v1.0).

## Upgrade from internal v2.x

If you were tracking Ulak OS's internal v2.x lineage, see `docs/runbooks/upgrading-from-v2.x.md` for the migration steps. Short version: `git pull`, re-run `ulak init .` in each project that uses Ulak OS, verify `bash scripts/validate-imports.sh` passes locally.

No breaking changes. The v1.0.0 semantic reset does not invalidate any prior artefact; the `@`-import chain is stable across v2.4.0 and v1.0.0 (public GA).

## Breaking changes

None from a runtime standpoint. All `@`-imports, commands, skills, agents, hooks, sector packs, rule packs, runtime rules, and governance docs continue to load unchanged. The semantic reset is in packaging and presentation: `version` string in three manifests (`package.json`, `prompts/pack.json`, `.claude-plugin/plugin.json`), documentation tone, and tag convention.

## Deprecations

- **Internal-release-notes archive** — files under `docs/archive/internal-releases/` describe operator-internal milestones (v1.5.0-V8, v1.6.0-V9, v1.7.0-V10.2, v1.8.0-V10.3) and remain for historical reference; they are NOT part of the public runtime surface and should be ignored by external consumers.
- **`UNLICENSED` license notice** — removed across all three package manifests. License is MIT.

## Known residuals (tracked, non-blocking)

- **CLI `src/` — empty** — The `src/` directory scaffold exists but has no implementation; `ulak` is a shell-script wrapper for now. A Node/Python CLI rewrite is tracked for post-GA v1.1.
- **`tests/` — empty** — no unit tests exist; `evals/` provides golden-set regression coverage. Unit tests tracked for post-GA v1.1.
- **Screenshots — placeholder** — `.claude-plugin/screenshots/` is a README directory. Operator captures ship in v1.0.1.
- **Plugin marketplace submission** — prep docs ready (`.claude-plugin/CATEGORIES.md`, `RATIONALE.md`); actual submission when Claude Code marketplace opens.
- **awesome-claude-code PR** — draft ready (`docs/distribution/awesome-claude-code-pr.md`); operator submits upstream when v1.0.0 tag is pushed to origin.

## Versioning forward

v1.0.0 → v1.0.1 (patch, screenshots + small fixes) → v1.1.0 (minor, first post-GA feature additions: CLI rewrite, tests) → v2.0.0 (post-GA major, alternative stack templates or LightRAG memory upgrade).

The internal v2.x lineage that preceded this release will not be repeated; all future majors bump from the v1.0.0 baseline forward.
