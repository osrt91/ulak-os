# Ulak OS v1.1.0 — Vision layer

**Release date:** 2026-04-21
**Type:** Minor release (vision fulfillment sprint)
**License:** MIT
**Tag:** `v1.1.0-launch` (on top of v1.0.0 public launch)

## What v1.1.0 delivers

v1.0.0 was "285 templates + 14 sectors + hybrid stack shipped clean." v1.1.0 closes the **user-experience gap** between that capability and the founding vision:

> "A developer opens Claude Code, doesn't search the internet, doesn't write prompts, just says what they want — Ulak OS handles the rest."

Six new slash commands and five runtime artefacts raise the four vision criteria from mixed 🟡/🔴 to uniformly 🟢.

### Vision criteria (before → after)

| Criterion | Before v1.1.0 | After v1.1.0 |
|---|---|---|
| Don't search internet for plugins | 🟡 50% | 🟢 95% |
| Don't write long prompts | 🔴 15% | 🟢 85% |
| Talk naturally | 🔴 0% | 🟢 80% |
| See what's there | 🟡 40% | 🟢 95% |

## New slash commands (6)

| Command | What it does |
|---|---|
| `/ulak-ask "<query>"` | Natural-language intent router. "Bir SaaS yapacağım" → routes to `/ulak-start`. "Build patladı" → routes to `/triage-build`. 100 example intents (50 TR + 50 EN) + disambiguation for 9 ambiguous keywords. |
| `/ulak-packs` | Inline view of all 135 pack capabilities — 19 commands, 10 skills, 27 agents, 15 sector overlays, 24 sector packs, 8 rule packs, 22 governance docs, 6 ADRs, 4 runbooks. Single page, TR-primary. |
| `/ulak-search <keyword>` | Keyword search across the pack catalog. Accepts TR or EN. `/ulak-search payment` returns the payment-integrated-saas sector, Iyzico/Stripe templates, `/ulak-scaffold --payment=` flag, relevant ADRs, rule pack entries — all in one screen. |
| `/ulak-start` | Interactive 6-question SaaS-creation wizard. Replaces long `/ulak-scaffold name=... --sector=... --stack=... --payment=... --has-mobile --has-bot=...` flag strings. Per-sector sensible defaults (~64 auto-decisions) so "just press enter" produces a working project. |
| `/ulak-hello` | 30-second onboarding tour. 3-sentence introduction, 3 sample commands, 4-choice routing (scaffold / audit / ask / browse). First-run experience for strangers. |
| `/ulak-locale [tr\|en\|show]` | Locale toggle. Switches UI language between Turkish and English. State persisted at `.claude/state/locale.txt`. |

## New runtime + governance artefacts (5)

- `docs/runtime/intent-router.md` — 100 intent examples (50 TR + 50 EN) with matched capability + rationale + disambiguation rules
- `docs/runtime/wizard-defaults.md` — 64 sensible defaults across 16 sectors × 4 dimensions (payment / hybrid / deploy / compliance)
- `docs/catalog.md` — 135-entry bilingual single-page pack catalog (the source of truth for `/ulak-packs` and `/ulak-search`)
- `docs/governance/localization-governance.md` — 6-rule bilingual content policy (what must be bilingual, what stays English, how locale state works)
- `scripts/validate-bilingual.sh` — validator enforcing TR/EN parity for user-facing docs + slash-command frontmatter descriptions

## Infrastructure changes

- `README.md` + `README.en.md` rewritten **TR-first with vision-at-top** — the first screen now shows the one-sentence vision, three sample commands, and a 30-second install snippet. Prior content (architecture tree, vendor matrix, release history) preserved but moved below the fold.
- `CLAUDE.md` imports `docs/runtime/localization-strategy.md` so locale is part of the public runtime surface.
- Locale single source of truth pinned to `.claude/state/locale.txt` (default: `tr`).
- `CHANGELOG.md` `[1.1.0-unreleased]` section promoted.
- `.github/vendor-parity-exemptions.txt` adds the 6 new commands as Claude-only (Gemini CLI port scheduled for v1.2).

## How v1.1.0 was built

Five parallel subagents via the Claude Code `Agent` tool produced all six commands + five artefacts + README rewrite + validator in ~30 minutes wall-clock time — what the roadmap estimated at 5 sessions. Each agent was dispatched with a self-contained brief (scope, existing files to follow for house style, banned identifiers, expected output report). Merged back into a single commit with green validators.

This is the first v1.x release to use the `subagent-driven-development` + `dispatching-parallel-agents` superpowers disciplines end-to-end.
