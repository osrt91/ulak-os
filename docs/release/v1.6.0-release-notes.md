# Ulak OS v1.6.0 — Cross-vendor parity

**Release date:** 2026-04-21
**Type:** Minor release (parity + trigger surface expansion)
**License:** MIT
**Tag:** `v1.6.0-launch`

## Summary

v1.6.0 closes the last vendor parity gap: every Ulak OS capability is now reachable from Claude Code, Gemini CLI, Codex CLI, and Copilot Chat. Where v1.5.0 shipped the catalog and wizard depth but Gemini CLI only understood 7 of the 24 slash commands, v1.6.0 generates all 24 as native `.toml` files, ships a 90-phrase natural-language trigger map for Codex, and a 60+ trigger expansion for Copilot Chat. The new `docs/governance/vendor-capability-matrix.md` is the authoritative primitive × vendor + command × vendor truth source — any future capability must update this matrix before it is considered "shipped."

## Vendor parity — before / after

| Vendor | v1.5.0 | v1.6.0 | Delta |
|---|---|---|---|
| Claude Code | 24/24 native | 24/24 native | — |
| Gemini CLI | 7/24 native | **24/24 native** | +17 `.toml` files generated |
| Codex CLI | ad-hoc NL | **24/24 NL trigger map** | +90 trigger phrases in AGENTS.md |
| Copilot Chat | unsupported | **22/24 NL trigger** | new adapter; 2 MCP-dep commands degraded |

The two degraded Copilot Chat commands are `/ulak-mcp-discover` and `/ulak-pattern-extract` — both depend on MCP primitives Copilot Chat does not currently expose. Graceful-degrade messages ship in `docs/adapters/copilot-chat.md`; Claude Code / Gemini CLI remain fully supported paths for those flows.

## Workstream details

### Workstream 1 — Gemini CLI native parity

`scripts/sync-gemini-commands.sh` (idempotent) reads `.claude/commands/*.md` and generates matching `.gemini/commands/*.toml`. Frontmatter from the Claude `.md` source is projected into Gemini's TOML shape (name, description, prompt). Running the script on a clean tree produces zero diff; running after a new command lands produces exactly one new `.toml`.

18 new .toml commands landed (v1.5.0 had 7, v1.6.0 ships 25 with the pre-existing director / final-verdict / intake / pack-gap-audit / market-scan / frontend/war-room files). Every top-level Claude command now has a Gemini twin.

### Workstream 2 — Codex CLI natural-language trigger map

Codex CLI does not execute slash commands; it executes natural-language intent. AGENTS.md now contains a 55-line "Natural-Language Trigger Map" section mapping 90 operator phrases (TR + EN) to the 24 Ulak capabilities. Example mappings:

- "audit projeyi" / "run a full audit" → `/director komple`
- "yeni saas başlat" / "start a new saas" → `/ulak-scaffold` via `/ulak-start`
- "eksik paket" / "what's missing" → `/pack-gap-audit`
- "paralel ajan dağıt" / "dispatch parallel agents" → `/ulak-subagent-dispatch`
- "mcp keşfet" / "discover mcp" → `/ulak-mcp-discover`

### Workstream 3 — Copilot Chat adapter

`docs/adapters/copilot-chat.md` (113 lines, new) documents how `.github/copilot-instructions.md` now expands to a 106-line NL Trigger Map with 60+ triggers. Copilot Chat cannot dispatch slash commands — but given the right trigger phrase in chat context, the loaded instructions drive Copilot to reproduce the Ulak director protocol inline. 22/24 commands work this way; 2 MCP-dependent commands carry a graceful-degrade message.

### Workstream 4 — Truth source + walkthroughs

`docs/governance/vendor-capability-matrix.md` (183 lines, new) is the single-source-of-truth for vendor × capability support. It encodes:

- **Primitive matrix** — slash / subagent / hook / MCP / file-write across 4 vendors
- **Command matrix** — all 24 commands × 4 vendors × (native / NL / degraded / unsupported)
- **Update rule** — any new capability must update this matrix in the same commit that ships the capability

Two walkthroughs shipped alongside:
- `docs/walkthrough/01-first-saas-end-to-end-codex.md` (671 lines) — first-SaaS flow from Codex CLI using only NL triggers
- `docs/walkthrough/01-first-saas-end-to-end-copilot.md` (360 lines) — same flow from VS Code + Copilot Chat

## How v1.6.0 was built

Four parallel subagents dispatched via `superpowers:dispatching-parallel-agents` in a single Claude Code turn, ~25 minutes wall-clock:

- Agent 1 — `sync-gemini-commands.sh` rewrite + 18 `.toml` generations
- Agent 2 — AGENTS.md NL Trigger Map (55 lines, 90 phrases)
- Agent 3 — `.github/copilot-instructions.md` expansion + `copilot-chat.md` adapter + 2 walkthroughs
- Agent 4 — `vendor-capability-matrix.md` authoring + adapter-contract updates (`codex-cli.md`, `gemini-cli.md`, `universal-runtime-contract.md`, `catalog.md`)

Final test + polish + CHANGELOG + release notes run as a fifth agent in parallel with user-manual bilingual update (TR + EN, separate workstream).

This is the largest parallel-dispatch v-cycle Ulak OS has exercised. The `superpowers:verification-before-completion` discipline gated every agent: no workstream closed until its validator leg was green.

## Upgrade path

No breaking change. `git pull && git checkout v1.6.0-launch` is the full operator path.

Fresh installs unchanged. `bash scripts/install.sh` and `iwr install.ps1 | iex` honor the new version automatically.

For existing Gemini CLI users on v1.5.0, new `.toml` files land under `.gemini/commands/` on pull; no user action required.

For Codex CLI and Copilot Chat users, reload the instructions file (`AGENTS.md` / `.github/copilot-instructions.md`) in the respective tool to pick up the new NL trigger maps. No config-file edits required.
