# Ulak OS v3.0.0 — Public launch consolidation

**Release date:** 2026-04-21
**Type:** Major release (consolidation of v1.1 → v2.2.1 into canonical v3)
**License:** MIT
**Tag map:** `v3.0.0` + alias `v3.0.0-public-ga` (push-safe; both point at the same HEAD)

## Summary in one paragraph

Between v1.0.0 public-GA and today, Ulak OS grew from a governance-skeleton scaffolder (27 templates, auth helper + RLS + CI + deploy + hardening) into a **full-stack product scaffolder**: 285 templates, 14 materialized sector overlay kits, hybrid systems (FastAPI backend + Expo mobile + Telegram bot + pnpm-workspaces monorepo + Traefik edge + k8s + ArgoCD), community ecosystem wrappers (6 new commands + 2 new skills), and a security hardening pass that closed 4 Critical + 9 High findings from an internal red team. v3.0.0 is the canonical label for that consolidation plus one hardcore history scrub pass that guarantees **zero leaked portfolio identifiers and zero leaked secret prefixes in any SHA**.

v3.0.0 does not introduce new headline features beyond what v1.1 → v2.2.1 already shipped. It is the public launch of the consolidated product.

## What v3.0.0 contains that v1.0.0 did not

### 1. Scaffolder is now a full product skeleton (not a skeleton of a skeleton)

v1.0.0 shipped 27 templates — auth helper, RLS baseline, CI workflow, deploy script, hardening template. Walkthrough text promised a 41-file tree; 14 files had to be synthesized at scaffold time. v3.0.0 closes the gap with **76 saas-starter templates** covering:

- **Auth flows** — login, register, forgot-password, layout, real Supabase SSR integration
- **Customer surface** — authenticated shell + dashboard + 5-tab settings
- **Admin surface** — shell with AP-06 fresh-DB role re-read, user CRUD, audit log viewer with filter/sort/pagination
- **Partner (reseller) surface** — commission dashboard, sub-user management
- **Marketing surface** — landing, about, pricing (monthly/annual toggle), blog + `[slug]`, changelog, docs
- **API routes** — public/health, customer/me, admin/users, partner/sub-users, webhooks/stripe, webhooks/iyzico
- **Real payment integration** — Stripe + Iyzico adapters, provider-agnostic interface, FSM (`pending → active → past_due → canceled`), webhook idempotency via `processed_webhook_events`, TRY + USD dual-column schema
- **15 shadcn primitives + 10 dashboard UI components** — button, input, form, dialog, sheet, tabs, table, data-table, sidebar, top-bar, empty-state, theme-provider
- **Dark mode + RTL + accessibility** — `next-themes`, Tailwind logical properties, focus-visible rings, prefers-reduced-motion, skip-to-content
- **i18n** — 100 TR + 100 EN keys (auth + nav + dashboards + errors + payment)
- **Docker Compose + Traefik + nginx + Dockerfile** (multi-stage, non-root `nextjs` user)
- **Tests** — auth e2e, Stripe webhook idempotency unit, Iyzico unit, RLS tenant isolation

### 2. Hybrid systems scaffolding

v1.0.0 could not produce multi-service products. v3.0.0 generates them in one scaffold call:

- **`templates/backends/fastapi/`** (24 files) — FastAPI async + SQLAlchemy 2.0 + Alembic + Pydantic v2; `auth.py` verifies Supabase JWT via JWKS (RS256 pinned in prod); routes for health, customer, admin, webhooks; pytest + httpx + conftest
- **`templates/apps/mobile-expo/`** (16 files) — Expo Router v4 + Zustand session + TanStack Query; Supabase auth shared with web; EAS build profiles (dev/preview/production); bundle identifier + permissions in `app.json`
- **`templates/apps/bot-telegram/`** (15 files) — aiogram 3.x + FSM + MemoryStorage; handlers for start/help/settings; HMAC-signed service-to-service calls with `{signature, timestamp, nonce, body_hash}` and `InMemoryNonceStore` for replay prevention
- **`templates/monorepo-root/`** (8 files) — pnpm workspaces + Turborepo; hybrid `docker-compose.hybrid.yml`; Traefik edge with strict CSP (`default-src 'self'`, no `unsafe-eval`, `form-action 'self'`, `object-src 'none'`, `base-uri 'self'`)
- **`templates/shared-packages/{shared,ui,config}/`** (12 files) — Zod schemas (frontend + backend single source of truth), shared types, constants, `cn()`, format helpers, design tokens, variants, base `tsconfig`

Scaffolder skill merges base → hybrid → sector → compliance → deploy overlay in deterministic order, writing `scaffold-log.yaml` for audit.

### 3. 14 sector overlay kits materialized

Each sector (under `templates/sectors/<sector>/`) adds schema + business logic + UX overlay on top of the base. DB triggers enforce anti-patterns at the database level — not a code comment, not a lint rule, an actual `CREATE TRIGGER` that raises on violation:

| Sector | DB-enforced anti-pattern |
|---|---|
| education | COPPA consent trigger (`enforce_parental_consent_before_lesson_access`) |
| fintech | AML flag immutability + KYC state-machine progression |
| ecommerce | AP-EC-02 oversell prevention via `reserve_stock_on_order_item` trigger |
| marketplace | Commission immutability trigger after dispute-window close |
| enterprise-b2b | AP-SSO-01 IdP-trust enforcement (SAML `wantAuthnRequestsSigned: true`, signing order STE; OIDC client secret read only from `ALLOWED_SECRET_PREFIX`) |
| media-content | Publishing FSM `draft → review → published` with reviewer separation |
| health-sensitive | PHI envelope encryption (per-row DEK, KEK in KMS); break-glass audit immutability |
| ai-copilot | Google API key in `x-goog-api-key` header (not URL); cost attribution per user + tier |
| pwa-desktop | Service worker offline-first, background sync, install prompt |
| admin-cms-hardening | Impersonation four-eyes DB constraint (approver ≠ requester, TTL + audit) |
| ai-relay-cost-control | Atomic `ai_relay_reserve_budget` RPC (no TOCTOU snapshot); per-user/tier/day cap + fallback |
| member-gated-community | Posts read/insert inherit thread tier gate via `tier_rank` subquery |
| self-hosted-supabase | Kong + GoTrue + Realtime + Storage + Studio docker-compose + backup/restore runbook |
| regulated-saas | GDPR/KVKK/SOC 2 runbooks + data residency enforcement + audit retention policy |
| container-k8s | Kubernetes manifests (StatefulSet + Service + Ingress + HPA) + Helm chart + ArgoCD |

Sector compatibility matrix is **machine-readable** at `docs/runtime/sectors-compat.yaml` with `recommended` / `allowed` / `refuses` per sector (e.g., `container-k8s` refuses `self-hosted-supabase` without explicit override; `health-sensitive` recommends `regulated-saas`).

### 4. Security hardening (v2.2.1 batch, inherited into v3.0.0)

| ID | Severity | Fix |
|---|---|---|
| SEC-V22-C-01 | Critical | Iyzico V3 HMAC-SHA256 over raw body, `timingSafeEqual`, `iyziPaymentId` required, legacy classic sig opt-in |
| SEC-V22-C-02 | Critical | Bot HMAC: separate Bearer + HMAC secrets, `X-Timestamp` + `X-Nonce` + `X-Body-SHA256` headers, 60s window |
| SEC-V22-C-03 | Critical | FastAPI JWT: RS256 pinned in prod (HS256 only non-prod) via JWKS with TTL cache |
| SEC-V22-C-04 | Critical | SAML enforces `wantAuthnRequestsSigned: true`, `messageSigningOrder: STE` |
| SEC-V22-H-02 | High | OIDC `resolveSecret()` only accepts env vars with `ALLOWED_SECRET_PREFIX = 'OIDC_CLIENT_SECRET_'` |
| SEC-V22-H-04 | High | Community schema: posts inherit thread tier gate via `tier_rank` subquery (prevent tier bypass) |
| SEC-V22-H-06 | High | AI-copilot: Google API key moved from URL to `x-goog-api-key` header (prevents log + referrer leak) |
| SEC-V22-H-07 | High | AI-relay cost cap: atomic `ai_relay_reserve_budget` RPC replaces TOCTOU snapshot |
| SEC-V22-H-11 | High | Traefik CSP: drops `unsafe-eval`, adds `base-uri 'self'` + `form-action 'self'` + `object-src 'none'` |
| 26-file cartography P0 | — | Component stub gap closed (build no longer breaks on scaffold) |
| 3-file polish | — | saas-starter Dockerfile (multi-stage, non-root), `deploy.sh` COMMIT regex + `.next/` cleanup on 3 paths, `install-hooks.sh` HMAC-signed preflight-bypass with 24 h expiry + mint script |

## Hardcore history scrub (the reason v3.0.0 is tagged today and not yesterday)

After v2.2.1 the operator required one final pass: **zero traces of portfolio project names and leaked secret prefixes in any SHA, in any commit, in any branch, in the entire object database**.

Pass 1 (from SEC-INCIDENT-2026-04-21, v1.0.0 cycle) removed 7 banned portfolio project names from the working tree and indexed tree. Live-code grep returned 0 matches.

Pass 2 (this release) ran `git filter-repo --replace-text` a second time after discovering that 2 of the 7 portfolio names still appeared in the **diff history of commits that had fixed those same strings** — the `- "extracted from X"` lines of the fix diff itself still carried the pre-fix value. Pattern file and banned identifiers are intentionally not reproduced in this document to keep the scrub terminal; the operator retains the pattern file outside the repository.

Verification after pass 2:

```text
Banned portfolio names:        0 commits (via git log --all -S for each of the 7 names)
Leaked secret prefixes:        0 commits (Resend + Cloudflare values, 10-char prefixes)
Working-tree grep -i -E:       0 files   (all 7 names)
Object DB git log --all -S:    0 commits (all 7 names + 2 prefixes)
```

`ulakos` (the project's own name, lowercase) appears once as `director-komple-ulakos-self-audit` in `docs/governance/lock-file-hygiene.md` — a self-reference to Ulak OS, not a portfolio leak. `sk_test_` / `sk_live_` prefixes appear in template placeholders (`sk_test_dummy`, `sk_test_EXAMPLE...`, `sk_live_real`) — Stripe's standard prefix format used in tests and docs, no real keys. `cfk_...` and `re_...` appear in `docs/security/incidents/` as sanitized prefix examples documenting the leak event itself, no real values.

After pass 2 the backup branch was deleted, reflog expired, and `gc --aggressive --prune=now` ran. All prior annotated tags were automatically re-pointed by filter-repo to their rewritten commits. `v3.0.0` + `v3.0.0-public-ga` tags were placed at the rewritten HEAD.

## Push blocker (for anyone reading this in git log after the fact)

v3.0.0 is shipped **locally** as of the release date. Push to `origin` is blocked pending SEC-INCIDENT-2026-04-21 operator-side key rotation (Resend + Cloudflare). The dual-tag pattern (`vX.Y.Z` + `vX.Y.Z-*-alias`) was used through the v2.x cycle to keep origin's legacy tags intact during the incident window; v3.0.0 continues that pattern with the `-public-ga` alias. When rotation completes the operator pushes both tags with `--follow-tags` after confirming revocation of the old credentials.

## Install

No change from v1.0.0. Both one-liner installers continue to work:

```bash
# POSIX
curl -fsSL https://raw.githubusercontent.com/osrt91/ulak-os/main/scripts/install.sh | sh

# Windows PowerShell
iwr -useb https://raw.githubusercontent.com/osrt91/ulak-os/main/scripts/install.ps1 | iex

# Manual
git clone https://github.com/osrt91/ulak-os.git && cd ulak-os
```

After install, open Claude Code in any directory and run `/director komple` to audit or `/ulak-scaffold` to start a new SaaS.
