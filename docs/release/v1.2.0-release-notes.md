# Ulak OS v1.2.0 — Wizard deepening

**Release date:** 2026-04-21
**Type:** Minor release (depth expansion sprint)
**License:** MIT
**Tag:** `v1.2.0-launch` (on top of v1.1.0 vision layer)

## The headline

Between v1.1.0 and v1.2.0, one number changed: `/ulak-start` went from **6 questions to 21–27 questions** (branched per sector). The experience is still "press enter to accept defaults" — but the defaults now actually draw on the 14 sector overlay kits + 8 rule packs + 22 governance docs + ~50 anti-pattern catalog that Ulak OS already shipped.

In v1.1.0 the vision-layer commands existed as scaffolding. In v1.2.0 the wizard flow is deep enough to route a new SaaS through ~20 decision axes — auth strategy, database provider, storage, email, analytics, error tracking, backup cadence, team size, CI choice, deploy target, preview-per-PR, compliance, monitoring, region/data-residency, plus 3-4 sector-specific deep questions.

## What changed

### `/ulak-start` — 6 → 21–27 questions, 5 phases, branched

`.claude/commands/ulak-start.md`: 280 → 615 lines. Auto-dispatch on confirmation.

**Phase 1 — Project identity (4 questions)**
- Project name (slug, default generated from date)
- Sector (16 options: education, fintech, ecommerce, marketplace, enterprise-b2b, media-content, health-sensitive, ai-copilot, pwa-desktop, admin-cms-hardening, ai-relay-cost-control, member-gated-community, self-hosted-supabase, regulated-saas, container-k8s, minimal-saas, other)
- Audience tier (solo / small-team / SMB / enterprise / multi-tenant-SaaS — drives admin-hardening and SSO decisions)
- Language (tr / en / bilingual / multi-locale)

**Phase 2 — Tech stack (6 questions)**
- Stack (Next.js+Supabase default; hybrid add-ons: FastAPI backend, Expo mobile, aiogram Telegram bot)
- Auth strategy (email+password / magic link / OAuth: Google+GitHub+Apple / Auth.js SSO / all)
- Database (Supabase managed / self-hosted Postgres / hybrid)
- Payment provider (Stripe / Iyzico / both / none)
- Storage (Supabase Storage / S3 / Cloudflare R2 / local)
- Email / transactional (Resend / Postmark / AWS SES / SMTP / none)

**Phase 3 — Sector depth (0–4 questions, branched)**
Each sector unlocks its own deep questions. Examples:
- Ecommerce: inventory management? subscription products? digital goods?
- Fintech: KYC provider (Sumsub/Onfido/Veriff)? AML flow? BaaS licensed?
- Education: synchronous lessons? certifications? assessments? minor users (→ auto COPPA)?
- Health-sensitive: PHI encryption at rest (forced)? BAA required? telemedicine?
- AI-copilot: RAG vector DB? per-user cost cap? fallback model?
- Marketplace: commission rate default? escrow? dispute flow?
- Community: tiered membership? moderation queue? feed type?
- Enterprise B2B: SSO providers? SCIM? team-roles model?
- Media/CMS: publishing workflow? comment moderation? CDN?

**Phase 4 — Operations (5 questions)**
- Hosting region (EU / US / Turkey / Global — drives GDPR/KVKK auto-resolve)
- Analytics (Plausible self-host / PostHog / Mixpanel / GA4 / none)
- Error tracking (Sentry / Logflare / Axiom / none)
- Uptime monitor (Better Stack / Cronitor / self-host / none)
- Backup strategy (daily snapshot / continuous WAL / none — fintech + health forced to continuous-wal)

**Phase 5 — Team + deploy + compliance (6 questions)**
- Team size (solo / 2-5 / 6-20 / 20+ — drives CI policy and preview-deploy defaults)
- CI provider (GitHub Actions default / GitLab / custom)
- Deploy target (VPS+Traefik / k8s+ArgoCD / Vercel / Fly.io / self-hosted docker)
- Preview deploy per PR (yes/no — auto-yes when team-size > 1)
- Compliance (gdpr / kvkk / coppa / soc2 / hipaa / pci-dss — multi-select, auto-resolved by region + sector)
- Monitoring dashboard (Grafana self-host / Grafana managed / none — k8s deploy forces managed)

### Flow controls

- `/skip` — accept defaults for all remaining questions in the current phase
- `/back` — go one question back
- `/restart` — start over from Phase 1 Q1.1 (with confirmation)

### Auto-dispatch

The wizard produces a summary block showing every answer, the auto-resolved cross-cutting rules (e.g., "Region=tr added kvkk compliance automatically", "Team>1 enabled preview-per-pr"), and the final `/ulak-scaffold` command with all 14+ flags filled. User presses `[enter]` or `[e]` → Claude Code invokes `/ulak-scaffold` in the same turn. `[h]` cancels. `[d <question>]` jumps to edit a specific answer.

### `docs/runtime/wizard-defaults.md` — 57 → 280 lines

Default matrix coverage expanded from **64 entries** (16 sectors × 4 axes) to **285 entries**:

- General matrix: 16 sectors × 15 general axes = 240 entries
  (payment, hybrid, deploy, compliance, audience, language, region, team-size, auth, database, storage, email, analytics, error-tracking, uptime, backup, monitoring, ci, preview-deploy)
- Sector-deep: 15 sectors × 3 deep axes = 45 entries

**Cross-cutting rules: 6 → 17**. Examples:
- `region=tr` → auto-append `kvkk` to compliance
- `region=eu` → auto-append `gdpr`
- `sector=health-sensitive` → force `hipaa` + `phi-encryption=true`
- `sector=ai-copilot` → auto-bundle `ai-relay-cost-control` overlay
- `team-size > 1` → force `preview-per-pr=yes`
- `deploy=k8s` → auto-select `monitoring=grafana-managed`

## How v1.2.0 was built

Three parallel subagents, dispatched via the Claude Code `Agent` tool in a single message, ~20 minutes wall-clock. Each agent owned one workstream:

- Agent 1 — Wizard rewrite (615-line `/ulak-start` + 5-phase flow + auto-dispatch)
- Agent 2 — Default matrix expansion (280-line `wizard-defaults.md` + 285 entries + 17 cross-cutting rules)
- Agent 3 — Catalog sync + 15-command `description_en` backfill

This continues the v1.1.0 pattern of `subagent-driven-development` + `dispatching-parallel-agents` as a first-class build discipline.
