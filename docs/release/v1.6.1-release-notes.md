# Ulak OS v1.6.1 — Branding refresh + dependency hygiene

**Release date:** 2026-04-22
**Type:** Patch release (branding + housekeeping, no runtime contract change)
**License:** MIT
**Tag:** `v1.6.1`

## Summary

v1.6.0 shipped the last vendor parity piece — every Ulak OS capability is now reachable across Claude Code / Gemini CLI / Codex CLI / Copilot Chat. v1.6.1 is the public-surface finish pass that followed: a unified visual identity (orange palette + bilingual marker + social preview banner), a dependency hygiene batch, and a cleanup of stale lineage artefacts from the pre-public semver cycle. **No runtime contract, agent, command, or pack behaviour changes.** If v1.6.0 works for you, v1.6.1 works identically — just looks and reads cleaner on the outside.

## What shipped

### 1. Hero redesign — README (TR + EN)

Both `README.md` and `README.en.md` replaced the early linear layout with a centred hero, a Mermaid architecture diagram, and a 3-card "Where do I start?" CTA grid. The goal: someone landing on the repo in ten seconds can see **what it does**, **which CLIs it supports**, and **which of three starting paths fits them**.

- **Centred layout** — name, tagline, vendor badges, bilingual toggle, license + maintainer — above the fold without scrolling on a 1440×900 screen.
- **Mermaid architecture** — vendor fan-in (4 CLIs) → router → Phase 0-5 pipeline → 13-artefact output. Renders natively on GitHub with no external service.
- **3 CTA cards** — "Yeni kullanıcı" (new user tour), "Mevcut proje" (brownfield audit), "Yeni SaaS" (greenfield scaffolder). Each card links directly to the right `/ulak-*` entry command.
- **6-scenario 2×3 grid** — condensed "what can I do" map.
- **Capability summary tiles** — 24 commands / 10 skills / 27 agents / 14-dim audit / 8 rule packs / 22 governance docs / 33 anti-patterns / ~100 templates — live counts pulled from `prompts/pack.json`.
- **Release history table** — compact CHANGELOG excerpt keyed to release notes.

### 2. Orange palette unification

The first hero redesign used the indigo/violet palette (`#7c3aed` / `#1e1b4b` / `#4c1d95`) inherited from the early pack-generation era. That clashed with the new orange social-preview banner. v1.6.1 migrates the full public surface to a single warm orange palette:

| Token | Old (violet era) | New (orange era) | Usage |
|---|---|---|---|
| Primary accent | `#7c3aed` | `#ea580c` | Badge fill, Mermaid primary nodes |
| Dark label | `#1e1b4b` | `#431407` | Badge `labelColor`, gradient start |
| Stroke / deep | `#4c1d95` | `#7c2d12` | Mermaid stroke |
| Emerald success | `#10b981` | `#10b981` | Retained for artefact accent (contrast) |

Changed files: `README.md`, `README.en.md`, `docs/branding/README.md` (palette table rewritten with orange token set).

### 3. Social preview banner

`docs/branding/social-preview.png` (1280×640 PNG) — the Open Graph image GitHub shows when the repo URL is pasted into Twitter/X, Slack, Discord, LinkedIn, etc. Generated via `scripts/generate-social-preview.py` (Pillow 12 + Segoe UI). The banner went through four iterations to reach the final:

- **v1** — indigo/violet gradient, solid composition, no bilingual marker.
- **v2** — switched to warm orange gradient; added decorative concentric rings + orbit ellipse; faint inline bilingual pill added ("Türkçe · English" single-line).
- **v3** — bilingual marker promoted to two separate accent-coloured chips ("TR Türkçe" warm red + "EN English" amber); decorations pushed to corners to stop crowding the wordmark.
- **v4 (final)** — bilingual chips consolidated back into **one symmetric pill** (`Türkçe · English` with colour-graded tokens); **all chip text migrated to `anchor='mm'` geometric centering** to eliminate sub-pixel drift from manual bbox math; title uses `anchor='mm'` for both halo + shadow so offset stays clean under any font swap.

The generator script is idempotent and re-runnable — `python scripts/generate-social-preview.py` produces the exact same bytes given the same fonts. Regenerating after a palette tweak or layout change does not drift the output.

### 4. Bilingual identity — TR · EN

Until v1.6.1 the bilingual nature of the project (TR-first with full EN mirror across README, AGENTS.md, GEMINI.md, user manual) was only visible once you opened a document and spotted the `[**🇹🇷 Türkçe**] · [**🇬🇧 English**]` toggle. The banner now surfaces this at the *entry point*: a centred pill reading `Türkçe · English` sits directly above the wordmark, visible on every GitHub preview card and every social share.

Context: the v3.0.0 lineage was **abandoned** on 2026-04-21 in favour of the semver reset (v1.0.0-launch → v1.6.0 → v1.6.1). The tags were retained briefly as hidden anchors during the history scrub but served no public purpose. Removing them prevents confusion for anyone browsing `gh release list`.

## Upgrade notes

### npm-installed users

```bash
npm install -g ulak-os@1.6.1
# or, for a project-local pin:
npm install ulak-os@1.6.1
```

No breaking changes. CLI commands (`ulak init`, `ulak run`, `ulak memory search`) behave identically.

### Repo-cloned users

```bash
git fetch origin
git checkout main
git pull
```

No migration script needed. Re-run `scripts/init-claude.sh` or `scripts/init-gemini.sh` if you want the updated README rendered in your editor preview.

### Verifying the branding

Render the social preview locally:

```bash
python scripts/generate-social-preview.py
# writes docs/branding/social-preview.png
```

Upload to GitHub social preview (manual, one-time): `Settings → General → Social preview → Edit → Upload docs/branding/social-preview.png`.

## Meta — how v1.6.1 was built

Worked directly in Claude Code (Opus 4.7) under the user's solo-operator flow. The banner iteration cycle was human-in-the-loop — each version rendered, reviewed by the operator, revised on feedback ("turunculu daha şekil bir şeyler" → v2, "kaymış" → v4). Total wall-clock: ~4 hours across two days, most of it visual iteration.

No subagent dispatch needed — branding is a single-surface concern and did not warrant the parallel specialist pattern.
